item 1：把c++当作语言的联邦

c++的四种语言特性：

  • C：c++基于c，块、statements、预处理器、内置数据类型、数组、指针等都来自c
  • 面向对象的C++: 带有类的C
  • Template C++
  • STL

item 2: 能用consts, enums, inlines尽量不用#defines

当用常量代替 #defines 的时候，有两个需要注意的点：

  • 定义常量指针：const char *const authorName = 'xxx'
  • 具体类的常量：通常c++需要你在使用任何量前先定义它，但是具体类的静态整型常量不需要

#defines 既不能用来定义具体类的向量，也不能用来提供任何类型的封装
如果你不想让人们获得你的整型向量的指针或引用，可以使用enum实现

item3：无论何时尽可能使用const

将数据声明为常量能够让编译器快速定位使用错误。const可以应用于任何范围的对象，函数参数、返回类型以及成员函数整体。

1
2
3
4

char greeting[] = "Hello";
const char *p = greeting; //non-const pointer, const data
char * const p = greeting; //const pointer, non-const data
const char * const p = greeting; //const pointer, const data

将成员函数设置为const的目的是识别哪些成员函数可以在常量对象上调用。

  • 逐位常量：一个成员函数是const当且仅当它不修改任何对象的数据成员
  • 逻辑常量：const成员函数可以对调用它的对象修改内部的一些数据位，但是只能以用户探测不到的方式
    编程时遵守逻辑常量规则

常量成员函数承诺从不更改其对象的逻辑状态，但是非常量成员函数没有这样的承诺

item4：确保对象在使用前被初始化了

总是在使用你的对象前初始化他们。

确保所有的构造器初始化了对象里的一切。

对象的数据成员在进入构造器的主体前就被初始化了。

对象数据初始化的顺序不可变，为了避免跨编译单元的初始化次序问题，建议使用 reference-returning 函数。

reference-returning: define and initialize a local static object on line 1, return it one line 2.

实例可以帮助理解：

进一步假设，客户决定创建一个 Directory 对象，用来放置临时文件：Directory tempDir( params ); 则此时初始化次序的重要性就显露出来了：除非 tfs 在 tempDir 之前先被初始化，否则 tempDir 的构造函数将会用到尚未初始化的 tfs. 但是由于它们都是定义于不同编译单元的 non-local static 对象，无法确定 tfs 会在 tempDir 之前先被初始化。

使用 reference-returning 可以解决这个问题：

class FileSystem {...}; //同前
FileSystem& tfs() //该函数用来替换tfs对象
{
	static FileSystem fs;
	return fs;
}

class Directory {...};
Directory::Directory(params)
{
	...
	std::size_t disks = tfs().numDisks();
	...
}
Directory& tempDir()
{
	static Directory td;
	return td;
}

这样做的原因是：c++保证函数内的 local static 对象会在“该函数被调用”时初始化。所以当用函数调用来代替直接访问 non-local static 对象，就可以保证所获得的那个 reference 将指向一个历经初始化的对象。

总结： 为了避免在初始化前使用对象，需要记住三个原则：

  • 手动初始化内置类型的非成员变量
  • 使用成员初始化列表初始化对象的所有部分
  • 为了免除“跨编译单元的初始化次序”问题，使用 local static 对象替换 non-local static对象
