条款 25：将 constructor 和 non-member functions 虚化

所谓 virtual constructor 就是某种函数，视其获得的输入，可产生不同类型的对象。virtual constructors 在许多情况下有用，其中之一就是从磁盘（或网络或磁带等）读取对象信息。

class NewsLetter{
private:
    static NLComponent *readComponent(istream& str);
    virtual NLComponent *clone() const = 0;
};
NewsLetter::NewsLetter(istream& str){
    while(str){
        components.push_back(readComponent(str));
    }
}

上边的 readComponent 就是一个 virtual constructor。

有一种特别的 vitual constructor 即 virtual copy constructor，该 virtual constructor 会返回一个指针，指向其调用者（某对象）的一个副本。

class NLComponent{

class TextBlock: public NLComponent{

这样每一个 TextBlock 都可以调用他自己的 clone，其他的子类也可以调用他们自己对应的 clone().

将 Non-Member Functions 的行为虚化： 写一个虚函数做实际工作，在写一个什么都不做的非虚函数，只负责调用虚函数，为了避免带来不必要的成本，还可以将非虚函数 inline 化。

条款 26：限制某个 class 所能产生的对象数量

假如某个类只应该有一个对象，最简单的限制这个个数的方法就是将其 constructor 声明为 private，这样每个人都没有权力创建对象。或者做一个约束，每次创建的时候都返回static的对象：

class Printer{
public:
    friend Printer& thePrinter();或者static Printer& thePrinter();
private:
    Printer();
    Printer(const Printer& rhs);
};
Printer& thePrinter(){
    static Printer p;
    return p;
}

上述代码有 3 个特点：

  • Printer class 的 constructor 为 private，可以压制对象的诞生
  • 全局函数 thePrinter 被声明为此 class 的一个 friend，使其不受 private constructors 约束
  • thePrinter 内含一个 static Printer 对象，所以只有一个 Printer 对象会被产生出来

如果不想将 thePrinter 加入全局命名空间，可以将其改为 Printer 的一个 static member function; 或者把 Printer 和 thePrinter 放进一个 namespace 内。

创建对象的环境： 还有一个直观的方法来限制对象的个数，就是添加一个名为 numObjects 的 static 变量，来记录对象的个数，当然这种方法在出现继承的时候会出现问题（一个 Printer 和一个继承自 Printer 的 colorPrinter 同时存在的时候，就会超出 numObjects 个数，这个时候就需要限制继承。

允许对象来去自由： 如果使用伪构造函数的话，会导致对象销毁后，无法创建新的对象，解决方法就是一起使用上面的伪构造函数和计数器。

template <class BeingCounted>
class Counted{
public:
    class TooManyObjects{};
    static int objectCount(){return numObjects;}
protected:
    Counted();
    Counted(const Counted& rhs);
    ~Counted(){ --numObjects; }
private:
    static int numObjects;
    static const size_t maxObjects;
    void init();                 //避免构造函数的代码重复
};

template<class BeingCounted>
Counted<BeingCounted>::Counted(){init();}

template<class BeingCounted>
Counted<BeingCounted>::Counted(const Counted<BeingCounted>&){init();}

template<class BeingCounted>
void Counted<BeingCounted>::init(){
    if(numObjects >= maxObjects)throw TooManyObjects();
    ++numObjects;
}

class Printer:private Counted<Printer>{
public:
    static Printer* makePrinter(); // 伪构造函数
    using Counted<Printer>::objectCount;
    using Counted<Printer>::TooManyObjects;
}

条款 27：要求（或禁止）对象产生于 heap 之中

如果需要让某种类型的对象有 “delete this” 的能力，则必须要让该类型的对象被分配于 heap 中。如果需要保证某一类型绝不会发生内存泄漏，则要让该对象不从 heap 中产生。

要求对象产生于 heap 之中

为了实现这样的限制，必须找出一种方法，阻止 clients 不得使用 new 以外的方法产生对象。通常只需要让那些隐式调用的构造动作和析构动作不合法即可，最简单的办法即让 destructor 成为 private，而 constructor 为 public 。具体来说就是可以导入一个伪析构函数来调用真正的析构函数。

限制 destructor 和 constructors 的运用会带来另一个问题即妨碍了继承和内含。解决继承问题，可以让基类的 destructor 成为 protected（仍保持其 constructors 为 public）。而对于必须内含的类，可以修改为内含一个指针，指向原本内含的对象。

判断某个对象是否位于 heap 内

程序的地址空间以线性序列组织而成，其中 stack（栈）高地址往低地址成长，heap（堆）由低地址往高地址成长，而 static objeccts 通常位于 heap 之下：

然而，他们究竟在哪里视系统而定，没有一个绝对具移植性的办法可以决定某对象是否位于 heap 内。

相对而言，判断 “指针的删除动作” 是否安全要简单的多，只需判断此指针是否由 new 返回即可，可以自行写一个 operator new。

禁止对象产生于 heap 之中

欲阻止用户直接将对象实例化于 heap 之中，只需要让用户无法调用 new 即可。可以重写 operator new 将其声明为 private，因为 new operator 是通过 operator new 进行工作的。

条款 28：Smart Pointers（智能指针）

所谓 smart pointers 是那些看起来、用起来、感觉起来都像内建指针，但提供更多机能的一种对象。smart pointers 由 templates 产生出来，可以利用 templates 参数表明其所指对象的类型。

Smart Pointers 的构造、赋值、析构

Smart pointer 的构造通常非常简单：确定一个目标物（通常是利用 smart pointer 的 constructor 自变量），然后让 smart pointer 内部的 dumb pointer 指向它。如果尚未决定目标物，就将内部指针设为 0，或是发出一个错误信息。此外，如果一个 smart pointer 拥有它所指的对象，它就有责任在本身即将被销毁时删除该对象，前提是这个 smart pointer 所指对象系动态分配而得。

以 by value 方式传递 auto_ptrs 往往是个非常糟的做法，只有当你确定要讲对象拥有权移转给函数的某个参数时，才应该以 by value 方式传递 auto_ptrs.若一定要以 auto_ptrs 作为参数，应采用 pass-by-reference-to-const 方式。当 smart pointer 被复制，或是身为赋值动作的来源端时，它会被改变。

smart pointer 的 destructor 通常看起来如下：

条款 29：Reference counting（引用计数）

引用计数有两个好处：

  • 简化 heap objects 周边的记录工作：当对象使用了引用计数，它就拥有了自己，一旦不再有任何人使用它，便自动销毁自己。
  • 允许多个等值对象共享同一实值，避免多次存储。

引用计数的实现：

产生一个 class 不仅存储引用计数，也存储它们追踪的对象值。

技巧： 将一个 struct 嵌套放进一个 class 的 private 段落内，可以很方便地让该 class 的所有 members 有权处理这个 struct，而又能禁止任何其他人访问这个 struct。

条款 30：Proxy classes（代理类）

代理类可以让一些原本十分困难或几乎不可能完成的行为得以实现：

(1) 实现多维数组

实现二维数组：在 Array2D 中将 operator[] 重载，令其返回一个 Array1D 对象。然后再对 Array1D 重载其 operator[] 令其返回原先二维数组中的一个元素。

template<class T>
class Array2D{
public:
    Array2D(int dim1, int dim2);
    class Array1D{
    public:
        T& operator[](int index);
        const T& operator[](int index) const;
    };
    Array1D operator[](int index);
    const Array1D operator[](int index) const;
};
Array2D<int> data(10, 20);
cout << data[3][6] //这里面的[][]运算符是通过两次重载实现的

Array2D 的用户不需要知道 Array1D class 的存在。
